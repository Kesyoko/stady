行ったこと：S3エラーの修正<br>
このエラーはMVPリリースまでにAWSを用いて画像投稿機能を実装した際に<br>
```
E, [2025-10-31T13:29:27.329826 #92] ERROR -- : [ActiveJob] [ActiveStorage::PurgeJob] [a91b837b-8506-466d-9683-96c67b7dade3] Error performing ActiveStorage::PurgeJob (Job ID: a91b837b-8506-466d-9683-96c67b7dade3) from Async(default) in 423.28ms: Aws::S3::Errors::AccessDenied (User: arn:aws:iam::268428820072:user/travel-memories-app2 is not authorized to perform: s3:ListBucket on resource: "arn:aws:s3:::travel-memories-app2" because no identity-based policy allows the s3:ListBucket action):
```
というエラーが出てしまったものを本リリース時に先延ばしにしたものです。<br>
上記エラーを日本語訳にすると<br>
```
E, [2025-10-31T13:29:27.329826 #92] ERROR -- : [ActiveJob] [ActiveStorage::PurgeJob] [a91b837b-8506-466d-9683-96c67b7dade3] ActiveStorage::PurgeJobの実行中にエラーが発生しました (Job ID: a91b837b-8506-466d-9683-96c67b7dade3) from Async(default) in 423.28ms: Aws::S3::Errors::AccessDenied (User: arn:aws:iam::268428820072: user/travel-memories-app2 は、リソース 「arn:aws:s3:::travel-memories-app2」 に対して s3:ListBucket アクションを実行する権限がありません。これは、s3:ListBucket アクションを許可するアイデンティティベースのポリシーが存在しないためです):
※DeepL.com（無料版）で翻訳しました。
```
とのことでしたので<br>
考えられる原因としては、画像を削除する権限かs3:ListBucketにアクセスする許可が与えられていなかったのが問題ではないか？と考えました。<br>
<br>
調べたこと：<br>
まず、用語がわからなかったので検索し技術記事を参考にし<br>
以下とまとめました。<br>
```
オブジェクト：ファイル1つ1つ。
バケット：全世界で一意の名前のS3内のファイルを管理する基本単位となるストレージ。
ポリシー：アクセス条件を設定するための仕組み
バケットポリシー：
バケットごとに細かくアクセス制御を実施したい場合に使う。アクセスされる側(S3バケット)にアタッチするポリシー(1個のみ)。
s3:DeleteObject ＝オブジェクト（ファイル）を削除するための権限。
s3:ListBucket　＝　バケット内のオブジェクト一覧の取得や存在確認などに必要な権限。
```
<br>
その上で調べを進め[【S3】オブジェクトを取得するIAMにs3:ListBucketも付与した方が良さそうって話](https://dev.classmethod.jp/articles/s3_getobject_listbucket/)の「CLI がオブジェクトの存在確認を行おうとした際に、s3:ListBucket 権限がないためです」の部分を参考にし進めることとしました。<br>
記事では名前をつけてアクセスしても良いものを列挙し、Resourceに/*をつけない形でバケット名を指定することでエラーが出なくなっていたのでそれに倣う形で記述しエラーが出なくなったので解決したと思われます。<br>
