#### ついに沼ると言われるRails基礎に突入しました😇<rb>
　
**基礎１学習したこと記録**
Railsのgコマンドとは？
ファイルやモデルを作成するためのコマンド。gはジェネレーターから来ている。
ついでに、ジェネレーターとは日本語に訳すと生成器。
　
docker compose exec〜 とは？
docker compose で定義されたコンテナ内でコマンド使いますよ！！という意味。
　
**具体的なコマンドについて**　
◇```docker compose exec web rails g model```は
docker compose execで定義した、名前がwebのコンテナ内でrails用の  modeを　g（ジェネレーター）を使って作ってねということ。app/model配下に作成。
　
◇```docker compose exec wed rails g migration```は
上記と同じようにマイグレーションを作ってねということ。migration配下に作成。
　
◇```docker compose exec web rails g controller```は
上記と同じようにコントローラーを作ってねということ。app/controller配下に作成。
　
◇コマンド違うけど、どっちもmigrationファイル作ってない？？？
どちらもマイグレーションファイルを作るが用途が違う。
〜modelは初めて作る際に必要。
〜migrationは作ったものを変更かけるときのためのマイグレーションファイルを作成するコマンド。
なお、双方ともコードを記入した後db:migrateを忘れないように注意。
　
◇使い方
```docker compose exec web rails g model ```を実行してしてモデル作成
``` docker compose exec web rails g migration```で作成したものにコードを書き
``` docker compose exec web rails db:migrate ```で変更をマイグレート。
また同じDBに変更があれば ```docker compose exec web rails g migration ```でコードを書き変更を加え
再度```docker compose exec web rails db:migrate```で変更をマイグレートするを繰り返すというような流れ。同じdbに変更があるたび、繰り返す。
