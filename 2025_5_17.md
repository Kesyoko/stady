#### ついに沼ると言われるRails基礎に突入しました😇<br>
　<br>
**基礎１学習したこと記録**<br>
Railsのgコマンドとは？<br>
ファイルやモデルを作成するためのコマンド。gはジェネレーターから来ている。<br>
ついでに、ジェネレーターとは日本語に訳すと生成器。<br>
　<br>
docker compose exec〜 とは？<br>
docker compose で定義されたコンテナ内でコマンド使いますよ！！という意味。<br>
　<br>
**具体的なコマンドについて**　<br>
◇```docker compose exec web rails g model```は<br>
docker compose execで定義した、名前がwebのコンテナ内でrails用の  modeを　g（ジェネレーター）を使って作ってねということ。app/model配下に作成。<br>
　<br>
◇```docker compose exec wed rails g migration```は<br>
上記と同じようにマイグレーションを作ってねということ。migration配下に作成。<br>
　<br>
◇```docker compose exec web rails g controller```は<br>
上記と同じようにコントローラーを作ってねということ。app/controller配下に作成。<br>
　<br>
◇コマンド違うけど、どっちもmigrationファイル作ってない？？？<br>
どちらもマイグレーションファイルを作るが用途が違う。<br>
〜modelは初めて作る際に必要。<br>
〜migrationは作ったものを変更かけるときのためのマイグレーションファイルを作成するコマンド。<br>
なお、双方ともコードを記入した後db:migrateを忘れないように注意。<br>
　<br>
◇使い方<br>
```docker compose exec web rails g model ```を実行してしてモデル作成<br>
``` docker compose exec web rails g migration```で作成したものにコードを書き<br>
``` docker compose exec web rails db:migrate ```で変更をマイグレート。<br>
また同じDBに変更があれば ```docker compose exec web rails g migration ```でコードを書き変更を加え<br>
再度```docker compose exec web rails db:migrate```で変更をマイグレートするを繰り返すというような流れ。同じdbに変更があるたび、繰り返す。<br>
<br>
◇(*〇〇)<br>
グループ化できる。<br>
<br>
config/application.rbファイルの下部に```config.generators```と記載があるところでジェネレーター設定をいじる。<br>
