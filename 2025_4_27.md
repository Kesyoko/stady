２３日から実装フェーズに入るまでの間でエラーに苦しみました。<br>
そこで行ったことは<br>
<li>ロボらんてくんに聞いてみる</li>
<li>インターネットで検索する</li>
<li>過去の質問に同様のものがないかを確認する</li>
<li>質問コーナーで講師に聞いてみる</li>
<li>過去のカリキュラムを参考に自力でどうにかできないか調べ、実行する</li>
<li>技術面談を利用する</li><br>
ということを行いました。<br>
　
結果として窮地を救ってくれたのは技術面談でしたが質問コーナーも大いに参考になることがたくさんあったので以下に記録します<br>
<h3>なぜエラーが起きたか</h3>
私の場合、マイグレーションファイルがいなくなったりstatusがダウンになったりしたため。　
<h3>モデルを作るコマンド</h3>
```
docker compose exec web rails generate model モデル名
```
これは新しくモデルと対応するマイグレーションファイルを制作するためのコマンド。
　
マイグレーションを実行する

docker compose exec web bin/rails db:migrate
モデルやマイグレーションファイルを作成し、必要なことを記述した後にコマンドを使用すると実行される
 
状態を確認する

docker compose exec web bin/rails db:migrate:status
これで今何のファイルがいるか確認できる。

これらを一通り実行し、モデルをカリキュラム通りに作成した後に再読み込みとサーバーをdocker compose upで再起動をかけたら直りました！！！！
