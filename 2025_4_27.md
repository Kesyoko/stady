<h3>なぜエラーが起きたか</h3>
私の場合、マイグレーションファイルがいなくなったりstatusがダウンになったりしたため。<br>
<h3>何を使用したか</h3>
<h4>モデルを作るコマンド</h4>
docker compose exec web rails generate model モデル名<br>
これは新しくモデルと対応するマイグレーションファイルを制作するためのコマンド。
　
<h4>マイグレーションを実行する</h4>
docker compose exec web bin/rails db:migrate<br>
モデルやマイグレーションファイルを作成し、必要なことを記述した後にコマンドを使用すると実行される<br>
 
<h4>状態を確認する</h4>
docker compose exec web bin/rails db:migrate:status<br>
これで今何のファイルがいるか確認できる。<br>
<h3>結果的に何を実行したらエラーが解消されたのか</h3>
これらを一通り実行し、モデルをカリキュラム通りに作成した後に再読み込みとサーバーをdocker compose upで再起動をかけたら直った。<br>

<h3>他に勉強になったことなど</h3>

respond_to do |format|とは何か<br>
これはリクエスト形式に応じたレスポンスを返すためのメソッド。クライアントが求めている形式で処理を実行することができる。<br>
例えば
respond_to do |format|
      if @name.update(name_params)
        format.html { redirect_to name_url(@name), notice: "名前の更新に成功しました" }
と書かれたプログラムであれば名前更新リクエストを受け取ってhtml形式でレスポンス(名前の更新に成功しましたの部分)を返している。<br>
　
updateは何する？<br>
updateは受け取った引数を更新する。<br>
書き方は＠〇〇.update(引数)。<br>
これで引数を受け取り＠〇〇をの値を更新する。<br>
　
上記とif~elseが組み合わさったら何ができる？<br>
もし、〇〇に引数があらかじめ決めてある形で保存できたら、〇〇という文字を表示してそれが成功しなかったら何も表示しないでね、ということができる。<br>

destroyは何をする？<br>
＠〇〇に入っている関数を消す。<br>
　
どうやって消したいものを特定している？<br>
コントローラー内でID（params[:id])を使って特定しているので可能となる。<br>

そもそもバリデーションってなんぞ <br>
バリデーションとは入力されたものが正しいかどうかを確認すること。<br>
例えば年齢入力欄に数字型を指定しておけば入力された値が文字型であった場合エラーを返す、などが可能となる。<br>
ついでにバリデーションの日本語訳は「確認」「検証」となるらしい。<br>
　
バリデーションの指定ってどうやるの？どこに何書いたらいいの？<br>
models内で定義する。自動生成された２行目に<br>
 validates :name, presence: trueと書いていく。これの意味は名前の入力欄に文字入力必須というルールを適用してね、ということ。<br>
validates :age, presence: true, numericality:{only_integer:true}と記載した場合、意味は年齢欄に入力が必須かつ数字は整数という条件を満たさないとダメということ。<br>
整数でという部分はnumericality:{only_integer:true}にあたり、<br>
numericality:で数値型かつonly_integer:trueで整数のという意味となる。<br>
　

そういえばコントローラーって何するんだっけ？<br>
操作が行われた際に処理を記述するところのこと。<br>
例えば名前の変更が行われた際に保存したり、リンクをクリックされたらページを表示するなど。<br>
　
paramsはなにするだっけ<br>
送られてきた値を受け取るためのメゾット。IDとか。<br>
